Ledger+gdrive

I’ve been dabbling around with a few personal finance tools, but never found anything that really satisfies my minimalistic, functional needs while I can still access it from everywhere. (H)Ledger is the right kind of functionality and minimalistic tool that I need, but gettting it distributed is a hassle. On my phone I’ve been using My Budget Book, which is a fantastic application, but it’s a hassle to get things synced and there is no way to get it on my computer.

So today I got enough of it and decided to figure out a simple way to work this out, my solution, 3 short bash scripts that could be 1 if I wanted. A pull and push script, and a script that calls them al in order.

pull-journal.sh is basically a oneliner that use rclone to sync my gdrive to my local folder.

#!/bin/bash
echo "Pulling journal from gdrive ..."
rclone -v sync <remote>:<path>/<to>/<file> /<path>/<to>/<journal-folder>

push-journal.sh is to same as pull-journal.sh but in the other direction.

#!/bin/bash
echo "Pushing journal to gdrive ..."
rclone -v sync /<path>/<to>/<journal> <remote>:/<path>/<to>/<journal>

Both come together in edit-journal.sh, which basicall pull the journal, then uses vim to edit it and afterwards pushes the journal back to my gdrive.

#!bin/bash
./pull-journal.sh
"Editing the file ..."
vim $LEDGER_FILE
echo ""
./push-journal.sh`

I kept the push and pull split in cases that I wanna force synchronisation in a certain direction.

On my mobile devices I just use some editor to modify the journal file. I can then later refactor it so it adheres to all the rules that (h)ledger require.
