DSCP

Current implementation for the ToS (Type of Service) field is split into 2 main parts: Express Congestion Notification (ECN) and Differentiated Services Codepoint (DSCP). This article focusses on the DSCP field.

In general you can use whatever DSCP value you like within you own network, but externally RFC 4594 describes how the DSCP value can be interpreted. The RFC specifies 3 kinds of classes:

  • EF - Expidited Forwarding: Reserved for voice traffic
  • AF - Assured Forwarding: Assured classed ranging from AF11 to AF43 (second number represents the drop probability)
  • CS - Class Selector: backwards compatible with the initial Precedence values

There’s a direct correlation between the numbers after AF, the DSCP value and the ToS. See the following example.

AF31    011(=3) 01(=1)
DSCP    011     01 0 = 26
ToS     011     01 0 00 = 104

The DSCP value is the first 6 bits of the ToS field, so the conversion between DSCP and the ToS field is <<2 and visa versa >>2.
